# PM Course Template

## Module 1: What This Product Actually Is

- Explain the product in user terms
- Reframe it as a runtime system, not just an interface

## Module 2: The Runtime Loop

- Show how a single request moves through the system
- Identify where state, tools, and model decisions interact

## Module 3: Capability Packaging

- Compare commands, tools, skills, plugins, and external integrations
- Explain what each layer is good for from a PM perspective

## Module 4: Trust And Recovery

- Show permissions, confirmations, policy checks, and recovery flows
- Explain why these are product features, not infrastructure details

## Module 5: Metrics And Operating Model

- Define completion, recovery, latency, cost, and governance metrics
- Explain how the team should monitor them

## Module 6: Design Workshop

- Ask learners to propose a new capability
- Require them to specify boundary, permissions, failure handling, and success metrics
