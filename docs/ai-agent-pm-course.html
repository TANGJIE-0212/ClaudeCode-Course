Claude Code for AI Agent PM

一份面向 AI Agent PM 的课程型讲解：它如何把“聊天模型”变成“可执行、可恢复、可治理”的工程代理，以及这些设计对 PM 的能力建设意味着什么。

course + deck
01 / Thesis

这不是一个聊天壳子，而是一个 AI Agent Runtime

从代码结构看，Claude Code 的核心不是单次问答，而是一个持续回合制的执行系统：装配上下文、调用模型、执行工具、处理权限、压缩上下文、恢复失败，再把系统状态稳定地带到下一轮。

对 AI Agent PM 来说，产品设计对象不只是“模型能力”，而是“模型 x 工具 x 策略 x 恢复机制”的复合系统。
02 / Architecture

核心模块如何分工

启动与装配

main.tsx 负责启动时序、配置加载、鉴权、特性开关、命令和技能注册、渲染器初始化。

代理主循环

query.ts 负责消息流、模型调用、工具回合、自动 compact、预算控制、恢复逻辑。

能力平台

tools.ts 和 Tool.ts 把读写文件、Bash、搜索、MCP、LSP、任务系统抽象成统一工具协议。

用户入口层

commands.ts 组织斜杠命令，承担“显式产品入口”的角色，比如 review、memory、tasks、config、doctor。

上下文与环境感知

context.ts 注入 git 状态、日期、CLAUDE.md 等上下文，让模型不是在真空里工作。

03 / Runtime Flow

一次用户请求是怎么跑完的

1

Startup

系统启动时并行预取配置、Keychain、远端设置和技能注册，减少冷启动等待。

2

Context Assembly

系统补齐 git 状态、CLAUDE.md、会话历史、模型设定、权限模式和工具清单。

3

Model Turn

模型流式返回内容，可能直接回复，也可能产出 tool_use，进入代理执行分支。

4

Tool Loop

工具执行结果被回灌给模型，直到本轮任务达到完成态、预算上限或恢复条件。

5

Recovery

如果上下文过长、输出超限或中途异常，系统会 compact、重试、摘要或切换收敛策略，而不是直接崩掉。

04 / PM Lens

对 AI Agent PM 的第一层启示

Agent 不是 Model Wrapper

Runtime

真正的产品竞争力来自运行时编排，而不是只会发一次 API 请求。

Trust 不是文案问题

Policy

权限模式、工具边界、用户确认点，才是用户愿不愿意放权的关键。

好体验靠恢复力

Recovery

AI 产品常失败，但能否可恢复、可解释、可继续，决定是否可用。

05 / Capability Design

从代码里看到的能力打包方式

Commands

  • 面向用户的顶层入口
  • 适合包装清晰任务，如 review、doctor、memory
  • PM 可把它理解为产品导航和任务模板

Tools

  • 面向模型的底层执行能力
  • 适合拆成可治理、可授权、可审计的原子操作
  • PM 可把它理解为 agent 的手和脚

Skills

  • 面向复用流程的知识打包
  • 适合把“会做”沉淀成“稳定能复现地做”
  • PM 可把它理解为可发现的最佳实践模板

Plugins / MCP / Bridge

  • 面向平台扩展和生态接入
  • 让 agent 从本地 CLI 走向 IDE、远端系统和团队协作
  • PM 要考虑生态边界、责任边界和授权边界
06 / PM Implications

AI Agent PM 该从这里学到什么能力

Capability Mapping

把用户意图拆到运行时层

不只写 PRD，还要知道一次请求会穿过哪些状态、工具、权限和恢复路径。

Guardrail Design

设计可放权的安全边界

决定哪些动作默认允许、哪些必须确认、哪些只能在特定模式下执行。

Failure Productization

把失败做成可恢复体验

失败不是异常分支，而是核心旅程。compact、重试、回退、摘要都应产品化。

Observability

定义你真正关心的 agent 指标

不是只有 DAU 和满意度。还要看任务完成率、平均工具回合数、权限拦截率、上下文压缩频次、恢复成功率、单任务成本。

Extensibility

把产品做成平台，而不只是功能堆叠

Command、Tool、Skill、Plugin、Bridge 是不同层的扩展位。PM 要知道新增能力应该落在哪一层，而不是全塞进一个 prompt。

08 / Closing

真正值得 PM 学的，不是 prompt，而是系统设计

这套代码告诉我们：AI Agent 产品的本质，是把模型放进一个有结构、有权限、有记忆、有恢复机制的执行系统里。PM 的工作，是定义这个系统如何可靠、可控、可扩展地完成用户任务。

如果你只能描述模型会什么，你还不是 AI Agent PM。你要能描述系统如何在现实环境里把能力交付出来。
