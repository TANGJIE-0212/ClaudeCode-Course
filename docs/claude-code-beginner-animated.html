第 1 章

Claude Code 是什么？

它不是一个简单的 AI 聊天框，而是一个完整的 AI 编程助手运行时。它能读代码、写代码、执行命令、管理记忆——像一个真正的程序员搭档。

🤖

不只是对话

普通 AI 只能回答问题。Claude Code 可以自主决定：先读哪个文件、用什么工具、怎么一步步完成任务。

🔧

8 个核心工具

读文件、写文件、编辑代码、搜索文件、搜索内容、执行命令、调用技能、启动子 Agent。

🔄

自动循环

不是一问一答，而是自动循环执行：调用 AI → 执行工具 → 把结果返回 AI → 继续…直到任务完成。

💡 一句话理解

把 Claude Code 想象成一个 会自己用工具的 AI 程序员。你告诉他「帮我加一个登录功能」，他会自己看代码、自己决定怎么改、一步步完成，最后告诉你「做好了」。

第 2 章

核心引擎：Agent Loop

这是整个系统的心脏。一个不断转动的循环：接收消息 → 调用 AI → 执行工具 → 返回结果，直到任务完成。

🧠

→
⚙️
AI 返回 文本 / 工具调用
→
🔧
执行工具 读写文件 / 命令
→
📋

agent.ts — 核心循环（简化版） 
// Agent Loop 核心：就是一个 while(true) 循环
while (true) {
  // 1. 压缩上下文（省 token，省钱）
  this.runCompressionPipeline();

  // 2. 调用 AI API（流式返回）
  const response = await this.callAnthropicStream();

  // 3. 看看 AI 有没有要调用的工具
  const toolUses = response.content.filter(b => b.type === "tool_use");

  // 4. 没有工具调用 → 任务完成，退出循环
  if (toolUses.length === 0) break;

  // 5. 有工具调用 → 检查权限 → 执行 → 把结果返回给 AI
  for (const toolUse of toolUses) {
    const perm = checkPermission(toolUse.name, toolUse.input);
    const result = await this.executeToolCall(toolUse.name, toolUse.input);
    // 结果返回消息历史 → 下一轮循环 AI 就能看到了
  }

  // 6. 检查预算（钱花完了就停）
  if (this.checkBudget().exceeded) break;
}

第 4 章

上下文管理：4 层压缩流水线

AI 的记忆窗口有限（200K tokens），聊多了就装不下。4 层压缩从最便宜的方式开始，逐级释放空间——前 3 层不花一分钱！

1

Budget 截断

超过 50% 上下文时，自动截断过长的工具结果。保留头尾，中间用「已截断」代替。

免费 🆓
2

Stale Snip

超过 60% 时，把重复读过的旧文件结果替换成「已省略，需要时重新读取」。

免费 🆓
3

Microcompact

空闲超过 5 分钟时，清理所有旧的工具结果——反正缓存也过期了。

免费 🆓
4

Auto-compact

上下文用到 85% 时，调用 AI 总结整段对话，压缩成一段摘要。

要花钱 💰
agent.ts — 压缩流水线 
// 每次调用 API 前，先跑一遍压缩
runCompressionPipeline() {
  this.budgetToolResults();     // 第 1 层：截断大结果
  this.snipStaleResults();      // 第 2 层：省略旧结果
  this.microcompact();          // 第 3 层：空闲时清理
}
// 第 4 层在 API 返回后检查：
if (inputTokens > window * 0.85) {
  await this.compactConversation(); // 调用 AI 总结压缩
}
第 5 章

权限与安全：不让 AI 搞破坏

AI 能执行命令是把双刃剑，必须管好权限。5 种模式 + 16 个危险命令正则，确保 AI 不会 rm -rf /。

🛡️

default

默认模式。危险操作需要用户确认

📋

plan

只读模式。只能看文件，不能写、不能执行命令

✅

acceptEdits

自动批准文件编辑，危险命令仍需确认

⚡

bypass

YOLO 模式！全部放行，危险但效率最高

🚫

dontAsk

CI 模式。需确认的操作直接拒绝

tools.ts — 危险命令检测 
// 16 个危险命令正则，覆盖 Linux + Windows
const DANGEROUS_PATTERNS = [
  /\brm\s/,           // 删除文件
  /\bgit\s+(push|reset)/, // git 危险操作
  /\bsudo\b/,         // 提权
  /\bdel\s/i,          // Windows 删除
  /\bRemove-Item\s/i,  // PowerShell 删除
  // ... 还有 11 个
];

// 权限检查三步走：
// 1. 先查配置规则（.claude/settings.json）
// 2. 再看当前模式
// 3. 最后检查危险模式
function checkPermission(tool, input, mode) {
  if (mode === "bypassPermissions") return { action: "allow" };
  if (ruleResult === "deny") return { action: "deny" };
  if (isDangerous(command)) return { action: "confirm" };
  return { action: "allow" };
}

第 7 章

记忆与技能：让 AI 记住你

每次对话都从零开始？不！记忆系统让 AI 跨会话记住你的偏好和项目信息；技能系统让你把流程模板化。

🧠 4 类记忆

User 记忆 — 用户偏好（如"喜欢中文注释"）
Project 记忆 — 项目信息（如"用 React + TypeScript"）
Feedback 记忆 — 纠正建议（如"不要用 var"）
Reference 记忆 — 参考文档（API 风格等）

🎯 技能系统

在 .claude/skills/ 目录放 Markdown 文件，就是一个技能。两种执行方式：

Inline 模式 — 把技能 prompt 注入到当前对话
Fork 模式 — 启动独立子 Agent 执行，不污染主对话

🔍

planning

子 Agent 运行原理 
// 主 Agent 调用子 Agent 就像调一个函数
const subAgent = new Agent({
  model: this.model,
  customSystemPrompt: config.systemPrompt,  // 专用提示词
  customTools: config.tools,                 // 受限工具集
  isSubAgent: true,                          // 标记为子 Agent
});

const result = await subAgent.runOnce(prompt);  // 独立运行
this.totalInputTokens += result.tokens.input;  // token 计入主 Agent
return result.text;                             // 结果返回主 Agent
第 9 章

架构全景：整体是怎么拼在一起的

从用户输入到任务完成，每一层各司其职，通过消息传递协作。

CLI 入口
cli.ts — 参数解析、REPL 循环、会话恢复、预算 flags
Agent 核心
agent.ts — Agent Loop、流式输出、4 层压缩、子 Agent 调度
工具层
tools.ts — 8 工具实现 + 5 权限模式 + 引号容错 + diff 输出
Prompt 层
prompt.ts — System Prompt 模板 + 记忆/技能/Agent 动态注入
扩展层
memory.ts / skills.ts / subagent.ts — 记忆系统、技能系统、多 Agent
持久化层
session.ts — 会话自动保存/恢复，支持 --resume 继续上次对话
